# App idea (edit or replace)

Describe the personalized mini-app in natural language:

- **Who** is the user and what context are they in?
- **What** screens or flows should exist?
- **Which** external APIs or data sources matter (if any)?
- **How** should it behave on mobile (portrait/landscape)?

Example: “A Google search mini-app: one search box, submit opens Google results for arbitrary keywords. Pin-friendly name: Google Search.”

When this file is present together with `mock_docs/`, UDAGen uses both. When only `--idea` is passed at generate time, the pipeline can still auto-draft mock API docs.
